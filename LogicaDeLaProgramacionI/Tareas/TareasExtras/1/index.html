Instrucciones

Cree una tabla comparativa en la cual compare las ventajas y desventajas de analizar un problema de las siguientes formas:

  • Con lenguaje natural
  • Por medio de un diagrama de flujo
  • Con pseudocodigo
  • Programando directamente

Tabla comparativa
Lenguaje Natural Diagrama de flujo Pseudocodigo Programando Directamente
Ventajas
  • Es sencillo
  • Es fácil de entender
  • Transmite rápidamente las ideas
  • Es intuitivo
  • Esta estandarizado
  • Puede describir mas que acciones
  • Sigue siendo legible por cualquier persona
  • Utiliza un lenguaje mas formal y estructurado
  • Se aproxima a la implementación
  • Para programas simples, puede convertirse a código directamente
  • Para programas simples, es mas rápido
  • Es directamente funcional
Desventajas
Es una idea muy general sobre lo que se quiere hacer. Los detalles de la implementación no son conocidos necesariamente. El proceso de realización puede ser pesado. Y también puede ser confuso si aun no tenemos una idea general de el algoritmo Puede que el lenguaje tenga algunas cosas que no tenga luego el lenguaje que vamos a usar para implementar el programa. Si no se tiene una idea general, o es un sistema mas complejo, trabajar directamente en código puede ser complicado, tardado, y puede dar unos resultados pésimos.
